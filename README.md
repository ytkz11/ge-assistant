# 我做了一个卫星地图助手，能看到同一个地方几十年的变化

你有没有好奇过，你家门前那条路十年前长什么样？你小区那块地以前是不是农田？你常去的那座山，卫星图上是什么样子？

我就好奇过。然后我就做了一个工具。

## 事情是这么开始的

我偶然发现 GitHub 上有个项目叫 GEHistoricalImagery，能从 Google Earth 和 Esri 下载历史卫星影像。但它是 C# 写的命令行工具，每次用都要敲一堆参数。

我想：这玩意儿要是能有个网页界面，拖一拖、点一点就能看到几十年前的卫星图，岂不是更爽？

于是说干就干。

## 它能干什么

这个工具叫 **GE Satellite Assistant**，简单来说就三件事：

**看历史卫星图。** 切到 Esri Wayback 来源，点一下查询，它会告诉你这片区域从 2022 年到现在有哪些可用的卫星影像。选一个日期，地图上就叠上那个时间点的卫星图。看着同一个地方从荒地变高楼、从农田变商场，挺震撼的。

**框选下载。** 你觉得某块区域有意思，框一下，选个缩放级别，它会告诉你这块区域有多少瓦片、多大尺寸、大概多大文件。确认之后就下载。支持 PNG 和 GeoTIFF 两种格式，TIF 带地理坐标信息，能直接丢进 ArcGIS 或 QGIS 里用。

**叠加中文标注。** 默认就开着高德的中文道路标注，卫星图上直接能看到路名、地名，不用自己猜这是哪儿。

## 踩过的坑

说几个印象深刻的。

**Esri Wayback 瓦片地址怎么找？** 起初我照着别人的思路用了错误的 URL 格式，瓦片死活加载不出来。后来去翻 Esri 的 WMTS capabilities XML，才搞明白瓦片 URL 的正确拼法——原来 `{z}` 对应的不是标准的 Web Mercator zoom，而是 WMTS 的 TileMatrix。调了一下午才对上。

**框选矩形消失的问题。** 最开始用 Leaflet 原生的 L.Rectangle 做框选，结果矩形在地图缩放时直接消失了。排查半天发现是事件监听的清理逻辑把矩形也清掉了。后来换成纯 DOM 覆盖层来画矩形，缩放时用地理坐标重新算像素位置，才搞定。

**Google 地球影像有水印。** 这个到现在也没完全解决。Google Earth 的历史影像瓦片经过 XOR 加密，需要先拿到 dbRoot 里的密钥解密。我参考 GEHistoricalImagery 实现了部分解密流程，但浏览器环境下的 CORS 限制让完整流程变得很麻烦。目前的方案是：看历史图用 Esri Wayback（无水印），下载也优先走 Esri。

## 技术栈

很轻量：

- **Vue 3** + **Vite**，前端框架
- **Leaflet**，地图渲染
- **Esri Wayback WMTS**，历史卫星影像数据源
- **geotiff** 库，导出带地理坐标的 TIF 文件
- **fast-xml-parser**，解析 WMTS capabilities XML
- 部署在 **GitHub Pages**，零成本

整个项目代码量不大，核心逻辑集中在几个文件里。如果你想魔改，很容易上手。

## 写在最后

这个工具最初就是满足我自己的好奇心——想看看某个地方以前长什么样。做到后来发现还挺有用的：做城市规划研究的同事拿它看土地变迁，搞环境监测的朋友用它对比植被变化，还有人拿它看自己老家二十年来的变化。
